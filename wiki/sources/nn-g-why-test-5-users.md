# Why You Only Need to Test with 5 Users

## Summary
Elaborate usability tests are a waste of resources. The best results come from testing no more than 5 users and running as many small tests as you can afford. This principle, known as the "discount usability engineering" approach, shows diminishing returns after testing 5 users.

## Key Takeaways
- Testing with 5 users finds approximately 85% of usability problems
- Each additional user provides diminishing returns in problem discovery
- Multiple small tests (test → fix → retest) are better than one large test
- The mathematics shows: N(1-(1-L)^n) where L is typically 31% problems found per user
- First user gives biggest jump (~31% insight), then diminishing returns
- After 5 users, the curve flattens significantly (85% problems found)

## Origins/History
Based on research by Jakob Nielsen and Tom Landauer showing the mathematical relationship between number of test users and usability problems discovered. This principle became a cornerstone of NN/g's "discount usability engineering" methodology.

## Applications
This principle is applied by UX professionals to:
- Optimize usability testing resources and budget
- Implement iterative design processes with continuous validation
- Reduce risk of atypical user behavior skewing results
- Focus on fixing discovered problems rather than endless testing
- Make testing accessible to projects with limited resources
