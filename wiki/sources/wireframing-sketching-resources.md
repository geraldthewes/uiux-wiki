# Wireframing and Sketching Resources
## Summary

A collection of authoritative resources on best practices for wireframing and sketching UI screens, focusing on low- to mid-fidelity approaches that emphasize structure, user flows, and functionality over visual polish.

## Key Takeaways
- Start with paper or whiteboard sketches before digital tools
- Focus on structure, hierarchy, and user flows — ignore colors, images, and branding early
- Use consistent conventions (rectangles for images, lines for text, etc.)
- Design for actual device constraints (thumb reach and 44px targets on mobile; responsive breakpoints on web)
- Test and iterate early with stakeholders/users
- Annotate clearly for developers and teammates
- Keep it low-fidelity until structure is validated

## Origins/History
Compiled from multiple authoritative UX sources including Nielsen Norman Group, Interaction Design Foundation, Smashing Magazine, Figma, and Balsamiq.

## Applications
- Early-stage design exploration and ideation
- Communicating layout and structure to stakeholders
- Testing information architecture and user flows
- Rapid iteration on interface concepts
- Mobile and web interface design
