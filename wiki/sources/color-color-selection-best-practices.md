# Considerations to Select the Main Colors for an App

## Summary
This resource provides comprehensive guidance on selecting and structuring color palettes for web applications, emphasizing the importance of limited, systematic palettes over ad-hoc color choices. It outlines best practices for defining core color categories, generating systematic scales, ensuring accessibility, and documenting usage rules.

## Key Takeaways
- **Limited palettes preferred**: 4-10 core base hues with 5-10 shades each (50-150+ total tokens)
- **Core categories**: Primary (1), Secondary/Accent (1-2), Neutrals (gray family), Semantic (4-5: success, warning, error, info)
- **60-30-10 rule**: 60% neutrals, 30% secondary, 10% accent colors for balanced interfaces
- **Selection process**: Start with brand primary, choose harmonious secondaries, build strong neutrals, define semantic colors, generate scales, test accessibility, document usage rules
- **Accessibility critical**: WCAG 2.2 AA minimum 4.5:1 contrast for normal text, 3:1 for large text
- **Design tokens recommended**: Use Figma Variables + CSS custom properties for consistent implementation

## Origins/History
This resource compiles best practices from multiple authoritative sources including Refactoring UI, EightShapes, Rohan Kamath, UX Design Collective, Interaction Design Foundation, and major design systems like Material Design 3, Carbon Design System, Ant Design, and USWDS.

## Applications
- Selecting brand-aligned primary colors
- Creating harmonious secondary/accent color relationships
- Building accessible neutral grays (avoiding pure black/white)
- Defining semantic colors for error states, success messages, warnings, and info
- Generating systematic shade/tint scales using HSL color space
- Testing and ensuring WCAG compliance
- Creating maintainable design token systems
- Planning for dark mode implementation
