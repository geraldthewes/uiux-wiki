# Hick's Law

The time it takes to make a decision increases with the number and complexity of choices.

## Mathematical Formula
RT = a + b * log2(n + 1)
Where:
- RT = Reaction time
- a, b = Constants determined empirically
- n = Number of equally probable alternatives

## UX Applications
- Menu design: Fewer, well-organized options improve decision speed
- Form design: Progressive disclosure reduces cognitive load
- Navigation: Limit top-level menu items (typically 5-9 based on Miller's Law)
- Checkout flow: Minimize steps and form fields
- Onboarding: Break complex processes into smaller steps

## Design Implications
1. Limit choices in menus and navigation
2. Use categorization and grouping to reduce perceived complexity
3. Implement progressive disclosure for complex interfaces
4. Use smart defaults to reduce decision-making burden
5. Consider alphabetical or logical ordering when appropriate
6. Test with time-on-task metrics to validate improvements
