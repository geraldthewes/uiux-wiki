# How to Draw a Wireframe (Even if You Can't Draw)
## Nielsen Norman Group

**Summary**
This article from Nielsen Norman Group provides a step-by-step guide to sketching wireframes, emphasizing that artistic ability is not required for effective low-fidelity wireframing. It covers conventions for representing common UI elements and offers practical techniques for focusing on structure and user flows.

**Key Takeaways**
- Messy sketches are acceptable and even encouraged for low-fidelity work
- Use constraints like thick pens or time-boxing to avoid getting bogged down in details
- Simple conventions: rectangles for images, lines for text, basic shapes for buttons and navigation
- Consider device-specific factors: aspect ratios, device chrome, and navigation patterns for web vs. mobile
- Related concepts: wireflows (wireframes + user flows), prototypes vs. wireframes, stakeholder sketching techniques

**Origins/History**
Nielsen Norman Group is a leading UX research and consulting firm founded by Jakob Nielsen and Don Norman, known for evidence-based usability guidance.

**Applications**
- Early-stage design exploration
- Communicating layout and structure to stakeholders and developers
- Testing information architecture and user flows
- Rapid iteration on interface concepts
