# Apple Human Interface Guidelines (HIG) Overview

The Apple Human Interface Guidelines provide in-depth information and UI resources for designing apps that integrate seamlessly with Apple platforms.

## Key Sections
- **Foundations**: Core principles of design (clarity, deference, depth), architecture, user interaction, and more.
- **Patterns**: Common interface patterns for navigation, modal experiences, etc.
- **Components**: Reusable UI elements like buttons, controls, views, etc.
- **Inputs**: User input methods including touch, keyboard, voice, etc.
- **Technologies**: Integration with Apple technologies like Siri, AR, etc.

## Platform-Specific Guidance
- **iOS**: Touch targets (minimum 44×44 pt), navigation patterns, tab bars, etc.
- **macOS**: Window management, menu bars, touch bar considerations.
- **watchOS**: Glances, notifications, complications, limited screen real estate.
- **tvOS**: Focus modeling, remote navigation, top shelf.
- **visionOS**: Spatial input, 3D objects, immersive experiences.

## Key Principles
- **Clarity**: Text is legible at every size, icons are precise and lucid, adornments are subtle and appropriate.
- **Deference**: The UI helps users understand and interact with the content, but never competes with it.
- **Depth**: Visual layers and realistic motion convey vitality and enhance understanding.

## Accessibility
- Support for Dynamic Type, VoiceOver, Switch Control, etc.
- Minimum touch target size: 44×44 pt
- Contrast ratios and color blindness considerations
