# Why You Only Need to Test with 5 Users

Jakob Nielsen, Nielsen Norman Group
March 18, 2000

## Summary
Elaborate usability tests are a waste of resources. The best results come from testing no more than 5 users and running as many small tests as you can afford.

## Key Findings

### The Mathematics of Usability Testing
- The number of usability problems found follows the formula: N(1-(1-L)^n)
  - N = total number of usability problems in design
  - L = proportion of usability problems discovered while testing a single user (typically 31%)
  - n = number of test users
- With 1 user: ~31% of problems found
- With 5 users: ~85% of problems found
- With 15 users: ~99% of problems found

### Why 5 Users is Optimal
1. **Diminishing returns**: After 5 users, additional testing yields minimal new insights
2. **Cost-effectiveness**: Better to run 3 studies with 5 users each than 1 study with 15 users
3. **Iterative design**: Enables multiple test-redesign cycles rather than one large study

## Benefits of Multiple Small Tests
1. **Problem fixing**: After finding 85% of problems with first 5 users, you can fix them and test again
2. **Quality assurance**: Second test validates that fixes worked and finds remaining issues
3. **Deeper insights**: Later tests can probe fundamental structure rather than just surface problems
4. **Risk mitigation**: Reduces chance of being misled by atypical user behavior

## When to Test More Users
Test additional users only when you have:
- Several highly distinct user groups (e.g., children and parents)
- Different user categories with sufficiently different behaviors

For multiple user groups:
- 3-4 users from each category for two groups
- 3 users from each category for three or more groups

## Practical Recommendations
1. Start with 5 users for initial testing
2. Fix discovered usability problems
3. Test again with 5 different users
4. Repeat as needed based on resources and complexity
5. Always test with representative users from your target audience
